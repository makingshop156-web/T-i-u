1. MDM vs Configuration Profile — Phân tích chuyên sâu

1.1 MDM là gì?

MDM (Mobile Device Management) là giao thức quản trị thiết bị di động của Apple, cho phép quản lý iOS từ xa qua server. MDM sử dụng APNs (Apple Push Notification Service) để gửi lệnh đến thiết bị.

1.3 Khi nào cần MDM?

  • Supervised-only restrictions: Một số restriction (allowAppRemoval, allowEnterpriseAppTrust, ...) chỉ hoạt động khi thiết bị supervised qua MDM
  • Zero-touch deployment: Cài profile tự động mà không cần người dùng thao tác
  • Cập nhật từ xa: Thay đổi restriction mà không cần gỡ/cài lại profile
  • Device query: Xem danh sách app, chứng chỉ, thông số bảo mật

1.4 Kết luận MDM

Đối với người dùng cá nhân muốn tối ưu hiệu năng, Configuration Profile bao phủ ~90% những gì MDM có thể làm. 10% còn lại (supervised-only keys, remote query, auto-update) không đáng để đầu tư server MDM + APNs certificate. Profile cài thủ công 1 lần là đủ.

2. REAL Device Measurements

2.1 WebGL GPU Benchmark — Cơ chế thực tế

WebGL (Web Graphics Library) là JavaScript binding của OpenGL ES, hoạt động trên iOS Safari từ phiên bản 8.0. Khi WebGL context được tạo, iOS cấp quyền truy cập GPU thông qua ANGLE layer (Almost Native Graphics Layer Engine).

JavaScript → WebGL API → ANGLE (translation layer) → OpenGL ES 3.0 → GPU driver → hardware

Đo FPS thực tế: Sử dụng requestAnimationFrame callback đồng bộ với v-sync (60Hz trên iPhone). Mỗi frame render, ta ghi nhận timestamp, tính delta, và FPS = 1000/delta.

for 180 frames: render(WebGL rotating cube) // REAL GPU work timestamp = performance.now() fps = 1000 / (timestamp - lastTimestamp) track average/min/max fps

Kết quả là REAL: FPS đo được phản ánh chính xác khả năng GPU của thiết bị, bị ảnh hưởng bởi:

  • GPU clock hiện tại (thermal throttle sẽ giảm FPS)
  • Shader complexity (vertex/fragment processing)
  • Fill rate (pixel throughput)
  • CPU main thread load (WebGL gọi qua JavaScript)

2.2 RAF CPU Profiler — Cơ chế thực tế

requestAnimationFrame (RAF) là Web API được thiết kế để đồng bộ hóa JavaScript animation với màn hình refresh. Trên iOS, RAF callback được gọi mỗi 16.67ms (60Hz).

Bằng cách đo thời gian giữa các RAF callback, ta có thể suy ra tải main thread:

  • Delta ~16.67ms: Main thread rảnh, frame budget đủ
  • Delta > 16.67ms: Main thread bận, frame bị trễ
  • Delta > 50ms: Jank — frame bị drop rõ rệt (thermal throttle hoặc daemon heavy)
CPU Load Estimation: avgDelta = average(frame_deltas) load = clamp((avgDelta - 2ms baseline) / 16.67ms * 100, 0, 100) Ví dụ: avgDelta = 20ms → load ≈ (18/16.67)*100 = 108% → clamp → 100% (main thread overloaded)

Jank %: Phần trăm frame có delta > 16.67ms. Jank > 15% cho thấy main thread đang quá tải.

2.3 DNS Multi-Domain Probe — Cơ chế thực tế

Test DNS blocking bằng cách gửi fetch request đến 5 ad/tracker domains với mode: 'no-cors'. Nếu AdGuard DNS đang chặn, request sẽ fail (network error). Kết quả:

2.4 Network Speed Test — Cơ chế thực tế

Download một resource nhỏ (Cloudflare favicon ~10KB) với cache: 'no-store', đo thời gian tải, ước tính throughput:

Mbps = (fileSizeBytes * 8) / (timeMs / 1000) / 1_000_000

Chạy 3 lần, lấy trung bình. Kết quả Mbps là REAL network throughput từ thiết bị đến CDN.

2.5 Battery Drain Tracking

Lưu battery level + timestamp vào localStorage. Khi user quay lại, so sánh với giá trị trước đó để tính drain rate:

drainRate = (levelBefore - levelNow) / (timeElapsedHours) // % per hour

Kết quả là REAL battery drain rate, chỉ hiển thị khi đủ thời gian giữa 2 lần đo (>5 phút).

4. Lợi ích Tổng hợp cho Game

Tác động đến gaming FPS:

1. CPU headroom ↑8-15%: 28 daemon không chạy nền = CPU có thêm xung nhịt cho game logic
2. RAM ↑100-200MB: Giảm memory pressure → iOS Jetsam không kill app nền → game swap ít hơn
3. Nhiệt ↓: CPU nền không sinh nhiệt → SoC giữ xung cao lâu hơn → FPS ổn định
4. Micro-stutter ↓: Daemon không đánh thức CPU đột ngột → frame time consistent
5. DNS blocking: Game không tải ad content → tiết kiệm CPU + bandwidth + giảm latency

5. Hạn chế REAL Measurement trên iOS

  • Battery API deprecated: Apple deprecated navigator.getBattery() trên iOS Safari 16+. Có thể không hoạt động ở iOS mới
  • Device Memory: navigator.deviceMemory chỉ hiển thị trên một số thiết bị/phiên bản
  • WebGL: FPS thấp hơn native game do WebGL driver overhead (ANGLE layer)
  • RAF CPU: Chỉ đo main thread, không measure GPU hay background threads
  • Network: Speed test dùng resource nhỏ (~10KB) — không chính xác bằng download file lớn
  • Storage: navigator.storage.estimate() chỉ ước tính storage của browser origin
